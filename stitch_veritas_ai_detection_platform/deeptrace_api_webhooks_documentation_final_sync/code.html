POST /v1/forensics/analyze

Media Analysis

The primary entry point for deep-media inspection. This endpoint ingests binary data or hosted URLs and returns a forensic confidence score across multiple detection vectors, including GAN residuals, frequency anomalies, and metadata inconsistency.

Request Body

media_url STRING REQUIRED

The publicly accessible URI of the image or video file. Supports .jpg, .png, .mp4, and .mov.

sensitivity INTEGER OPTIONAL

Detection threshold (1-100). Default is 75.

Response Schema

PROPERTY TYPE DESCRIPTION
request_id UUID Unique identifier for this forensic session.
confidence FLOAT Normalized score (0.0 - 1.0) of authenticity.
V2.4_GUIDE / Integration

Webhooks

Register asynchronous callback URLs to receive forensic scan results in real-time. Eliminate polling and react instantly to detected manipulations.

integration_instructions REGISTRATION STEPS

  • 01

    Deploy a publicly accessible HTTPS endpoint on your infrastructure.

  • 02

    Register the URL in the Developer Dashboard or include it in the API request header.

  • 03

    Implement signature verification using the X-DeepTrace-Signature header.

Retry Logic

Max Retries 12 Attempts
Window Span 24 Hours

After 12 failed attempts, webhooks are marked as "Failed". Use exponential backoff for best performance.

Request Example 
const deepTrace = require('deeptrace-sdk');

const result = await deepTrace.forensics.analyze({
  media_url: "https://cdn.intel/v/392.mp4",
  sensitivity: 85,
  priority: "high"
});
